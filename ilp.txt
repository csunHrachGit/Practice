Create a test repository, and work with your group members to do the activities:

Create a practice repository for your group on Github
Everyone create your own branch <your_dev_branch_name> and switch to it
$ git branch <your_dev_branch_name>
$ git switch <your_dev_branch_name>
Add some files to your own branch 
$ git add <file_name>
Commit your changes in your own branch
$ git commit -m "<commit_message>"
Push your own branch to the remote repository
$ git push -u origin <your_dev_branch_name>
Modify the files in the main branch, then merge main into your branch
$ git switch <your_dev_branch_name>
$ git merge main
Modify files in your own branch, then merge your branch back to the main branch and push main branch to the remote repository
$ git switch main
$ git pull
$ git merge <your_dev_branch_name>
$ git push -u origin main
Now you could practice with your group members
Add a version tag to your current branch
$ git tag -a v1.0 -m "this is version 1.0 for module A"
Keep editing files in your branch and commit your changes to the remote repository
To view the tags on the Github repository, use the command:
$ git push --tags
